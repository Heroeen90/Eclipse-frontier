## [0.1.0] - MVP Release

### الإضافات (Added)

#### نظام اللاعب (Player System)
- شخصية لاعب قابلة للحركة (8 اتجاهات) باستخدام `CharacterBody2D`.
- نظام إحصائيات اللاعب: الصحة (Health)، الطاقة (Stamina)، الخبرة (XP)، المستوى (Level).
- نظام رسوم متحركة (Idle, Walk, Run) مرتبط بحالة الحركة.
- نظام مستويات (Leveling System) مع زيادة تدريجية في الإحصائيات عند الترقية.

#### نظام الجرد (Inventory System)
- إضافة عناصر (Add Item) مع دعم التكديس (Stacking).
- حذف عناصر (Remove Item).
- تقسيم العناصر (Split Stack) إلى مكدّسات أصغر.
- واجهة جرد تفاعلية (Inventory UI) مرتبطة بالبيانات عبر Signals.
- حفظ واستعادة محتوى الجرد ضمن نظام الحفظ.

#### نظام الموارد (Resource System)
- أربعة أنواع موارد أساسية: خشب (Wood)، حجر (Stone)، حديد (Iron)، طعام (Food).
- نقاط جمع موارد (Resource Nodes) قابلة للتفاعل في خريطة العالم.
- مدير موارد مركزي (`ResourceManager`) للتحكم في الكميات والتحديثات.

#### نظام البناء (Building System)
- أربعة أنواع مبانٍ: منزل (House)، مخزن (Storage)، منجم (Mine)، مزرعة (Farm).
- نظام معاينة بناء (Builder Preview) قبل التثبيت النهائي.
- التحقق من توفر الموارد قبل السماح بالبناء.
- مدير مبانٍ (`BuildingManager`) لتتبع المباني المنشأة وحالتها.

#### نظام الحفظ (Save System)
- حفظ تلقائي بصيغة JSON يشمل: اللاعب، الموارد، المباني، المهام.
- نظام Checksum (SHA-256) لحماية ملفات الحفظ من التعديل اليدوي.
- التحقق من سلامة البيانات (Validation) قبل التحميل، مع رفض الملفات التالفة أو المعدَّلة.
- معالجة استثناءات شاملة عند فشل القراءة/الكتابة.

#### نظام المهام (Quest System)
- مهام رئيسية (Main Quests) ومهام جانبية (Side Quests).
- نظام مكافآت (Quest Rewards): موارد وخبرة عند إتمام المهمة.
- مدير مهام (`QuestManager`) لتتبع تقدّم وحالة كل مهمة (نشطة / مكتملة / غير مفعّلة).

#### واجهة المستخدم (UI System)
- القائمة الرئيسية (Main Menu).
- واجهة اللعب الأساسية (Gameplay HUD) تعرض الصحة، الطاقة، الموارد.
- واجهة الجرد (Inventory UI).
- واجهة البناء (Building Menu).
- واجهة المهام (Quest Menu).
- قائمة الإعدادات (Settings Menu).
- قائمة الإيقاف المؤقت (Pause Menu).
- مربع حوار تأكيد عام (Confirmation Dialog) قابل لإعادة الاستخدام.

#### النواة والبنية (Core / Architecture)
- نظام أحداث مركزي (`EventBus`) لفصل الأنظمة عن بعضها (Decoupling).
- نظام تسجيل أحداث (`Logger`) منظم بمستويات (Info, Warning, Error).
- معالج أخطاء مركزي (`ErrorHandler`).
- ملف ثوابت موحّد (`Constants`) لكل القيم الثابتة في المشروع.

#### الأمان (Security)
- فصل كامل بين بيانات اللعبة (Data Layer) والواجهة (UI Layer).
- تشفير أساسي لملفات الحفظ لمنع التعديل المباشر بمحرر نصوص.
- التحقق من صحة كل المدخلات الواردة من ملفات الحفظ قبل استخدامها (عدم الوثوق بالمدخلات).

#### الاختبارات (Testing)
- اختبارات أساسية لنظام اللاعب (`test_player.gd`).
- اختبارات أساسية لنظام الجرد (`test_inventory.gd`).
- اختبارات أساسية لنظام الحفظ (`test_save_system.gd`).
- اختبارات أساسية لنظام المهام (`test_quests.gd`).
- منفّذ اختبارات (`test_runner.gd`) لتشغيل جميع الاختبارات دفعة واحدة.

#### DevOps و النشر
- إعداد GitHub Actions (`android-build.yml`) لبناء APK تلقائيًا عند كل Push إلى `main`.
- ملف `export_presets.cfg` جاهز لتصدير Android APK (Debug).
- `.gitignore` مخصص لـ Godot 4.x.
- توثيق شامل (`README.md`, `ARCHITECTURE.md`, `SYSTEMS.md`, `API.md`, `TROUBLESHOOTING.md`).

### ملاحظات الإصدار

هذا الإصدار يمثل **النسخة الأولية القابلة للتشغيل (MVP)** من Eclipse Frontier، ويضع الأساس البرمجي والمعماري لكل التوسعات المستقبلية (نظام القتال، الأعداء، الزعماء، التجارة، تعدد اللاعبين).
